## Challenges

### Challenge #1 (PRIMARY): Responsive Layout = Bug Factory

**Problem:** 5 out of 6 production bugs in Story 5.3 were responsive-related. Radix Sheet + 3-tier layout (desktop/laptop/mobile) + Zustand state sync create combinatorial edge cases.

**Specific bugs found:**
1. Sheet not opening at laptop/mobile — `handleActiveFindingChange` only synced `selectedId` at desktop
2. Sheet width wrong — CSS specificity `sm:max-w-sm` overrode custom tokens
3. Touch target < 44px — FindingCardCompact row height below WCAG minimum
4. J/K at laptop = stale Sheet — keyboard nav didn't update `selectedId`
5. Mobile close → resize laptop = phantom Sheet re-open — `selectedId` not cleared on transition

**Root Cause:** Viewport transition logic scattered across 3 files (ReviewPageClient, FindingList, FindingDetailSheet). No centralized hook to manage selectedId/Sheet/drawer state across viewport changes.

**Impact:** Story 5.3 required 2 CR rounds specifically for responsive E2E fixes. 5 of 6 E2E test fixes were also responsive-related (selector changes, assertion mechanisms).

### Challenge #2: AI Pipeline Still Not Stable — Serious Bug Every Epic

**Problem:** Every epic since AI was introduced has had at least one serious AI pipeline bug:

| Epic | Bug | Severity | Days Undetected |
|------|-----|----------|----------------|
| Epic 3 | TD-AI-004: L2/L3 bracket bug — findings silently dropped | CRITICAL | 17 days |
| Epic 4 | TD-AI-005: CAS guard race — findings orphaned | HIGH | Until retro |
| Epic 4 | TD-AI-006: L3 segment filter excludes failed L2 chunks | HIGH | Until retro |
| Epic 5 | Story 5.1 CR R1: Cache key missing targetTextHash | CRITICAL | Caught in CR |

**Root Cause:** Real AI integration test coverage is still thin. `bt-pipeline.integration.test.ts` covers back-translation only. L2 `runL2ForFile` and L3 `runL3ForFile` — the core pipeline — still rely entirely on mocked AI responses. Mock tests pass even when real AI output format changes (proven by TD-AI-004).

**Impact:** The core product value ("AI-powered QA") has had broken or degraded functionality in every epic. Detection is improving (17 days → caught in CR) but the pipeline itself keeps producing bugs.

**Recommended Action:** Real L2/L3 integration tests (not just BT) that call real AI API + verify findings inserted into DB. This should be a P0 prep task for Epic 6.

### Challenge #3: Metadata Merge vs Replace Trap

**Problem:** Story 5.2a required merging `non_native` key into existing review_actions metadata. 4 actions already had metadata (`isManual`, `undoType`, `originalSeverity`). Writing `metadata: { non_native }` instead of merging would silently destroy existing data.

**Mitigation:** Story doc highlighted this as CRITICAL pattern with code examples. CR R1 caught remaining gaps.

### Challenge #4: 3-Table Atomic Transactions (5.2c)

**Problem:** Native reviewer workflow requires atomic transactions across findings + finding_assignments + review_actions (+ notifications non-blocking + audit_logs). State machine complexity remains high even after story split.

**Mitigation:** Guardrail #67 + transaction pattern documented. All transitions tested in RLS integration tests.

1. **Bugs don't decrease — but detection shifts left.** CR R1 catches most issues, E2E catches responsive integration bugs, RLS tests catch security gaps. The metric isn't "fewer bugs created" but "more bugs caught before production."

3. **Story split strategy is proven effective.** CR rounds dropped from avg 1.9 (Epic 4) to 1.4 (Epic 5). Story 5.2 split prevented a repeat of Story 4.2's 3-round complexity.

4. **Prep tasks remain highest-ROI investment.** Pattern proven across 5 consecutive epics. Every prep task directly prevented re-discovery or re-work during implementation.

5. **AI pipeline has had serious bugs every epic since introduction.** Detection is improving (17 days → caught in CR) but core L2/L3 pipeline still lacks real integration tests. BT integration test exists but L2/L3 rely entirely on mocks.

6. **100% retro action item follow-through is achievable.** Epic 5 achieved 6/6 action items + 6/6 prep tasks from Epic 4 retro. Previous retro was 3/3 + 6/6. The pattern is sustainable.
