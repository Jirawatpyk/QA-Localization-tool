## Challenges

### Challenge #1 (CRITICAL): Mock-Heavy Testing = Invisible Failures

**Problem:** Unit tests mock everything — AI responses, DB transactions, Realtime subscriptions. This created a blind spot where core features could fail silently.

**Evidence:** TD-AI-004 — L2/L3 findings silently dropped for 17 days (since Epic 3). Prompt produced `[uuid]` brackets, validation expected bare `uuid`. Every L2/L3 pipeline run "completed successfully" with 0 findings. No test caught it because:
- Unit tests mocked AI responses with pre-formatted data
- No integration test called real AI API + verified DB inserts
- E2E tests seeded findings directly, skipping pipeline

**Impact:** Product core feature "AI-powered QA" did not work. Users would see 0 AI findings and assume files were clean. This is a product integrity gap, not just a testing gap.

**Root Causes (4 categories):**

| Category | Description | Affected TDs |
|----------|-------------|-------------|
| Integration Gaps | Mock AI/DB, never test real flow | TD-AI-004, AI-005, AI-006 |
| Component Isolation | Test per-component, miss cross-component interaction | TD-E2E-018, E2E-013 |
| Real Data ≠ Test Data | Small test corpus misses language-specific edge cases | TD-AI-002, AI-003 |
| Realtime + Concurrency | Mock Realtime, miss tenant filter gaps | TD-TENANT-003 |

### Challenge #2: Story Complexity Creates Tech Debt Clusters

**Problem:** 26 TDs created in Epic 4, but 69% (18/26) concentrated in just 3 stories.

| Story | TDs | Reason |
|-------|-----|--------|
| 4.0 (Infrastructure) | 7 | Inherited Epic 3 TDs + E2E skip accumulation |
| 4.2 (Core Actions) | 5 | High complexity: 8-state lifecycle × optimistic UI × Realtime × audit × auto-advance |

**Story 4.2 specifically:** 3 CR rounds (only story to exceed 2), 4 production bugs found via systematic review. AC count was within ≤8 limit, but cross-AC interaction complexity was extreme.

**Insight:** AC count alone is insufficient to measure story complexity. Cross-AC interactions must be evaluated.

## Key Insights

1. **"0 incidents" ≠ "no problems"** — If you don't test the real thing, you can't know. Epic 3's "0 production incidents" actually hid 1 CRITICAL bug (TD-AI-004).

2. **AC count doesn't measure complexity** — Cross-AC interactions must be assessed. If any AC coordinates with >3 other ACs → must split story.

3. **Verification story is a proven safety net** — Story 4.8 caught TD-AI-004 before ship. Should be mandatory last story of every epic.

4. **"Probably passes" ≠ "actually passes"** — Proven live during this retro: 3 unit tests were failing that nobody knew about.

5. **Silent drop / silent success = most dangerous failure mode** — Pipeline must fail loud, not log + continue.

6. **Prep tasks remain highest-ROI** — 6/6 completed, pattern proven across 3 consecutive epics.
